How to use

A strong password in three taps.

This tool generates passwords using crypto.getRandomValues(), the same cryptographic primitive your bank uses for session tokens.

Nothing is sent to a server. The generation, the entropy calculation, even the crack-time estimate — all of it runs in your browser. We literally cannot see what you generate.

If you want to verify, open DevTools → Network tab and click "Regenerate". You'll see exactly zero requests.

  1. Pick a length

    Anything above 16 characters is overkill for everything except a few specific attack scenarios. Default to 20, you'll be fine.

  2. Choose character sets

    All four on for max strength. Drop symbols if the target site doesn't accept them. Drop lookalikes if you'll type it manually.

  3. Copy and store

    Paste it into your password manager immediately. The "save .txt" button is for batch generation and team rotations — don't leave passwords in plaintext files.

  4. Rotate if leaked

    Run the tool again. No history is stored, no "previous passwords" cache. Each generation is independent.
